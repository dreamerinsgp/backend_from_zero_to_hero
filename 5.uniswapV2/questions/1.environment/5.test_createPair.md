# Q1：测试 `createPair` 需要什么前提条件？

分两种场景：**本仓库自带的单元测试**（推荐先跑通）和 **在链上真实调用 `createPair`**。

---

## A. 跑本仓库里的 `createPair` 测试（`yarn test`）

| 前提 | 说明 |
|------|------|
| **依赖已安装** | 在项目根目录执行 **`yarn install`**（建议 **Node 16.x**，见 `questions/1.environment.md`；过新的 Node 可能导致 `ethereum-waffle` / ganache 相关依赖报错）。 |
| **合约已编译** | **`yarn compile`** 生成 **`build/`**（含 `UniswapV2Factory.json`、`UniswapV2Pair.json` 等）。`pretest` 会默认先编译。 |
| **无需真实代币合约** | 测试里 `createPair` 传入的是两个**固定地址**（`TEST_ADDRESSES`），用于验证 Factory 部署 Pair、事件与 `getPair`；**不要求**链上已部署 ERC20。 |
| **无需 RPC / 私钥** | 使用 **ethereum-waffle `MockProvider`**（内置本地区块链），不连测试网。 |

**命令：** `yarn test`（会执行包含 `createPair` 用例的 `test/UniswapV2Factory.spec.ts`）。

---

## B. 在测试网 / 主网上调用 `createPair`（集成/手工验证）

| 前提 | 说明 |
|------|------|
| **已部署 `UniswapV2Factory`** | 先有 Factory 合约地址（例如用 `scripts/deploy.js` 部署）。 |
| **两个 ERC20 地址** | `tokenA`、`tokenB` 必须是**该网络上已部署**的 ERC20；Factory 会按 `token0 < token1` 排序后创建 Pair。 |
| **账户与 gas** | 调用者钱包在该网络有 **原生币** 支付 gas。 |
| **业务约束** | 两地址不能相同、不能为零地址、该交易对尚未创建（否则 `PAIR_EXISTS`）。 |

---

## 小结

- **只验证合约逻辑**：满足 **Node + `yarn` + `yarn compile` + `yarn test`** 即可（前提 A）。  
- **在真实链上建池**：需要 **Factory 地址 + 两个链上 ERC20 + gas**（前提 B）。

---

## Q2：在测试网上自建并部署两枚代币，再 `createPair`

目标：**你自己部署两枚 ERC20**，拿到两个链上地址，再对已部署的 **Factory** 调用 **`createPair(tokenA, tokenB)`**。

### 合约来源

使用本仓库 **`contracts/test/ERC20.sol`**：继承 `UniswapV2ERC20`，构造函数 **`constructor(uint _totalSupply)`**，把全部供应量铸给 **部署者**（`msg.sender`）。

### 步骤概览

1. **`yarn compile`**，确保存在 **`build/ERC20.json`**。  
2. 在 **同一测试网**（与 Factory 相同 RPC / chainId）配置 **`.env`**：`RPC_URL`、`PRIVATE_KEY`，账户内有 **测试网 ETH**。  
3. **部署两枚代币**（二选一）：  
   - **脚本（推荐）**：`yarn deploy:tokens`  
     - 默认各部署 **1000000 * 10^18** 最小单位（名称/符号仍为合约里写死的 `Uniswap V2` / `UNI-V2`，两枚合约地址不同即可）。  
     - 可选环境变量 **`SUPPLY_A_WEI`**、**`SUPPLY_B_WEI`**（十进制整数字符串，单位为 **wei**）。  
     - 终端会打印 **`TOKEN_A=`**、**`TOKEN_B=`** 地址。  
   - **Remix**：上传/粘贴合约与依赖，选 **Solidity 0.5.16**，连测试网，**部署两次** `ERC20`，每次传入不同的 `totalSupply`（uint）。  
4. **部署 Factory**（若尚未部署）：`yarn deploy`，记下 Factory 地址。  
5. 调用 **`createPair`**（顺序任意，Factory 内部会按 `token0 < token1` 排序）：  
   - **脚本（推荐）**：在 `.env` 中设置 `FACTORY_ADDRESS`、`TOKEN_A`、`TOKEN_B`（以及 `RPC_URL`、`PRIVATE_KEY`），然后执行 **`yarn createpair`**。若该交易对已存在，脚本会打印已有 Pair 地址并退出。  
   - 或在前端 / 区块浏览器中直接调用 Factory 的 **`createPair(TOKEN_A, TOKEN_B)`**。  

### 与「使用别人部署的代币」的区别

- **自建两枚**：供应量、部署者持有的余额由你控制，适合完全本地实验。  
- **用已有代币**：省 gas，但需找到可靠合约地址并领水龙头（见下节，可选）。

### 注意

- **`createPair` 不要求**代币「有名」或不同符号；只要两个**不同地址**的 ERC20。  
- 若代币有**暂停转账、黑名单**等，可能影响后续向 Pair **转币 / `mint`**，建池前可在浏览器读合约。  
- 测试代币与主网资产隔离：**私钥仅用于测试账号**。
