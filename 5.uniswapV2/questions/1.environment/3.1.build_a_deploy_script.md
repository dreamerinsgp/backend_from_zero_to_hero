# 自建部署脚本的工作流（ethers / web3）

对应 `questions/3.how_to_deploy_this_contract.md` 中的「B. 自建脚本」一节。下面是推荐顺序，便于复现与排错。

---

## 1. 前置条件

1. 已完成 **`yarn compile`**，`build/UniswapV2Factory.json`（及需要时的 ERC20 等）存在。
2. 已选定测试网，并准备好 **RPC URL**、**带测试币的私钥或助记词**（勿提交到 Git）。
3. 在 Node 环境中安装 **`ethers` v4**（与本仓库测试用的 `ethereum-waffle@2` / 合约 ABI 习惯一致）或 **ethers v5**（API 略有差异，需对应调整 import）。

---

## 2. 工作流（按顺序）

### 步骤 A：加载编译产物

- 读取 `build/UniswapV2Factory.json`。
- 取出 **`abi`**（或 `abi` 字段，视导出格式而定）。
- 取出 **`evm.bytecode.object`**，若不含 `0x` 前缀，部署时拼接为 **`0x` + bytecode**。

### 步骤 B：连接网络

- 用 **`JsonRpcProvider(RPC_URL)`**（ethers）连接测试网。
- 用 **`Wallet(privateKey, provider)`** 或从助记词派生钱包，作为 **feeToSetter / 部署者**（部署者需有测试网原生币付 gas）。

### 步骤 C：部署 Factory

- 使用 **`ContractFactory(abi, bytecode, signer)`**。
- **`deploy(feeToSetterAddress)`**，其中 `feeToSetterAddress` 为构造函数参数（常用部署者地址）。
- **`await factory.deployed()`**（ethers v4）或 **`await factory.waitForDeployment()`**（ethers v5）拿到已部署实例与地址。

### 步骤 D：（可选）部署测试 ERC20

- 若测试网没有现成代币，可同样用 `build/ERC20.json` 的 abi + bytecode 部署 `contracts/test/ERC20.sol`，构造函数为初始供应量（与合约一致）。

### 步骤 E：创建 Pair

- 拿到 **Factory 合约实例** 与两个 **ERC20 地址** `tokenA`、`tokenB`。
- 调用 **`createPair(tokenA, tokenB)`**，等待交易确认。
- 用 **`getPair(tokenA, tokenB)`** 验证返回的 Pair 地址非零。

---

## 3. 配置与安全习惯

| 项目 | 建议 |
|------|------|
| 私钥 | 使用 `.env` + `dotenv`，并把 `.env` 加入 `.gitignore` |
| 网络 | 用环境变量区分 `RPC_URL`、`CHAIN_ID`，避免硬编码 |
| 重跑 | 脚本应可重复执行时考虑「已部署则跳过」或读入已保存的 Factory 地址 |

---

## 4. 与 Hardhat / Foundry 的关系

- **同一逻辑**：加载 artifact → deploy Factory →（可选 deploy ERC20）→ `createPair`。
- **差异**：Hardhat/Foundry 用各自 **`artifacts`** 路径与 `getContractFactory`，常把 RPC/私钥放在 **`hardhat.config`** 或 **`foundry.toml` + cast**。

更宏观的部署说明见 `questions/3.how_to_deploy_this_contract.md`。

## Q2：部署脚本放在哪里？

本仓库**没有**规定目录；常见做法是：

| 方式 | 路径示例 | 说明 |
|------|----------|------|
| 纯 Node + ethers | 仓库根目录下 **`scripts/deploy.ts`** 或 **`scripts/deploy.js`** | 与 `contracts/`、`build/` 并列，脚本内用相对路径读 `build/UniswapV2Factory.json`。 |
| Hardhat | **`scripts/deploy.js`** 或 **`deploy/001_factory.ts`** | 合约与编译由 Hardhat 管理时需单独配置；也可把本仓库合约拷进 Hardhat 项目。 |
| Foundry | **`script/Deploy.s.sol`** | 用 Solidity 脚本或 `forge script`，artifact 路径按 Foundry 约定。 |

**建议：** 新建 **`scripts/`** 目录放部署脚本；在根目录 **`package.json`** 里加 `"deploy": "node scripts/deploy.js"`（或 `ts-node`）方便执行。将 **`scripts/` 内的密钥占位说明**写进 README 即可；**不要把私钥写进脚本或提交到 Git**。

## Q3：使用 Sepolia 测试网时要准备什么？

**Sepolia** 的 chainId 为 **`11155111`**。部署与调用前请准备：

| 需要提供 | 说明 |
|----------|------|
| **`RPC_URL`** | Sepolia 的 HTTPS 端点（Alchemy、Infura、Ankr、公共 RPC 等）。脚本里作为 `JsonRpcProvider` 的连接串。 |
| **部署账户私钥** | 仅用于本地/CI：放在 **`.env`** 的变量里（如 `PRIVATE_KEY`），**勿提交**。该地址在 Sepolia 上需有 **Sepolia ETH**（通过水龙头领取）用于支付 gas。 |
| **`feeToSetter`（构造函数参数）** | 一个 **以太坊地址**（常用即部署者地址）。Factory 部署时传入。 |
| **（可选）两枚 ERC20 地址** | 若要做 `createPair`，需要 **已在 Sepolia 上部署** 的两个代币合约地址；否则需先部署测试 ERC20 再 `createPair`。 |

**不必**为「能部署」单独提供：Etherscan API Key（仅验证合约源码时用）、助记词（除非你选择用助记词代替单私钥派生钱包）。

**环境变量示例（名称可自定）：**

```bash
RPC_URL=https://sepolia.infura.io/v3/<YOUR_PROJECT_ID>
# 或 Alchemy: https://eth-sepolia.g.alchemy.com/v2/<KEY>
PRIVATE_KEY=0x...   # 仅示例，真实值放 .env 且勿提交
```

仓库内已提供 **`scripts/deploy.js`** 与 **`.env.example`**：复制 `.env.example` 为 `.env` 并填写后执行 **`yarn deploy`**（需先 **`yarn compile`**）。

## Q4：账户私钥怎么从 MetaMask（小狐狸）导出？

**安全提示：** 私钥等同于账户完全控制权。仅用于**本地开发 / 测试网**；勿对**主网大额资金**账户操作；导出后勿截图发群、勿提交 Git、勿粘贴到不可信网站。

### 步骤（桌面浏览器插件，界面可能随版本略有差异）

1. 打开 **MetaMask** 插件，解锁钱包。
2. 点击右上角 **⋮**（或账户图标）进入 **账户详情**（不同版本入口可能是「账户详情」「Account details」）。
3. 找到 **导出私钥**（**Export private key**）。
4. 输入 MetaMask **密码** 确认。
5. 复制显示的 **私钥**（通常是一串 `0x` 开头的十六进制）。这就是可写入 `.env` 里 `PRIVATE_KEY=` 的值（整串复制，含 `0x` 与否以你的脚本约定为准，多数 ethers 脚本接受带 `0x`）。

### 移动端

一般在 **设置 → 安全与隐私 → 显示私钥** 或账户详情里导出，同样需要密码/生物识别。

### 更稳妥的替代

- 使用 **仅放测试网资产** 的独立账户再导出私钥。  
- 或使用 **硬件钱包** / **Cast / Foundry** 的 `ledger` 交互，避免把私钥放进 `.env`（需工具链支持）。
