# Q1: 运行 / 构建本合约需要哪些版本？

## Yarn

- 使用 **Yarn Classic（1.x）**。仓库的 `yarn.lock` 为 **lockfile v1**，对应 Yarn 1。
- `package.json` 中 `packageManager` 字段指定 **yarn@1.22.22**；使用 **1.22.x** 即可。

## Node.js

- `package.json` 的 `engines` 写的是 `node >= 10`，仅表示历史兼容范围。
- **实际建议：使用 Node 16.x（如 16.20.x）**。在较新的 Node（18 / 20 / 24）上，`yarn install` 可能因旧版原生依赖（如 `sha3` 等）无法编译而失败，导致无法安装依赖、也就无法编译合约。

## Solidity 编译器（solc）

- 依赖中固定为 **`solc@0.5.16`**，Waffle 配置使用 `./node_modules/solc`，构建时使用 **Solidity 0.5.16**。

## 简要对照

| 工具 | 建议版本 |
|------|----------|
| Yarn | 1.x（Classic），例如 1.22.22 |
| Node | 16.x（推荐，便于依赖安装成功） |
| solc | 0.5.16（随 `yarn install` 安装） |

## 相关

- 安装依赖：`yarn`
- 编译合约：`yarn compile`（详见 `questions/2.how_to_build_the_contract.md`）
