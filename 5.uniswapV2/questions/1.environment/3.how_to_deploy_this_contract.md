# Q3: 如何把编译好的合约部署到测试网？

本仓库**没有**内置部署脚本，只有合约源码与 `yarn compile` 生成的 `build/` 产物（ABI + bytecode）。部署需自行用钱包 / 脚本连接测试网 RPC。

环境准备见 `questions/1.environment.md`，编译见 `questions/2.how_to_build_the_contract.md`。

---

## 1. 部署顺序（概念）

| 合约 | 说明 |
|------|------|
| **UniswapV2Factory** | 必须先部署。构造函数：`constructor(address _feeToSetter)`，传入可管理手续费权限的地址（常用部署者地址或多签）。 |
| **UniswapV2Pair** | **不要**单独部署。由 `Factory.createPair(tokenA, tokenB)` 通过 **CREATE2** 按需创建；每个交易对一对代币一个 Pair。 |

可选：若仅做流动性测试，还需两条链上 **ERC20** 代币合约地址（可用测试网水龙头代币或自建 `contracts/test/ERC20.sol` 的部署实例）。

---

## 2. 测试网前准备

1. **编译**  
   `yarn compile`，确认 `build/UniswapV2Factory.json` 等存在。

2. **RPC**  
   任选测试网（如 **Sepolia**、**Holesky** 等），从 Alchemy、Infura、公共 RPC 等获取 **HTTPS RPC URL**。

3. **账户与 gas**  
   使用 MetaMask 或脚本中的私钥对应地址，从该测试网 **水龙头** 领取原生币支付 gas。

4. **链 ID**  
   部署与后续交互必须在**同一条链**（同一 chainId）。

---

## 3. 常见部署方式（任选其一）

### A. Remix

1. 打开 [Remix](https://remix.ethereum.org/)，加载或粘贴 `contracts/UniswapV2Factory.sol`（及依赖），编译器选 **0.5.16**。
2. 在 **Deploy** 里连接钱包，网络选目标测试网。
3. 部署 `UniswapV2Factory`，构造函数参数填 `_feeToSetter` 地址。
4. 部署完成后，用 Factory 合约界面调用 **`createPair`**，传入两个 ERC20 地址。

### B. 自建脚本（ethers / web3）

1. 读取 `build/UniswapV2Factory.json` 中的 `abi` 与 `evm.bytecode.object`（部署 Factory 用完整 bytecode）。
2. 用带签名者的 provider 连接测试网 RPC，`ContractFactory` 部署 Factory，传入 `feeToSetter`。
3. 拿到 Factory 地址后，用 `factory.createPair(tokenA, tokenB)` 创建 Pair（需两枚代币已部署且地址正确）。

（若使用 **Hardhat / Foundry**，可把本仓库当作子模块或复制合约，在同一 solc 版本下写部署脚本；仍遵循上述 Factory → `createPair` 顺序。）

---

## 4. 部署后可用调用（简述）

- `factory.getPair(token0, token1)`：查询某对代币的 Pair 地址。  
- `factory.allPairs(i)` / `allPairsLength()`：枚举已创建的交易对。  
- Pair 上的 `mint` / `swap` / `burn` 等需按 Uniswap V2 文档与流动性逻辑操作。

---

## 5. 与主网 / 官方实例的区别

- 测试网部署的是**你自己的** Factory 与 Pair，**不是** Ethereum 主网上 Uniswap 已部署的那套地址。  
- 若只想**调用**官方协议，应使用官方文档给出的**已部署合约地址**，而不是本仓库部署流程的必选项。
