# Q1：本仓库核心合约有哪些功能？（core features）

`v2-core` 主要包含 **`UniswapV2Factory`**、**`UniswapV2Pair`**（继承 **`UniswapV2ERC20`**）及库合约。按模块概括如下。

---

## UniswapV2Factory（工厂）

| 功能 | 说明 |
|------|------|
| **部署交易对** | `createPair(tokenA, tokenB)`：用 **CREATE2** 确定性部署 `UniswapV2Pair`，`token0 < token1` 排序，登记 `getPair` 与 `allPairs`。 |
| **手续费接收方** | `feeTo`：协议费相关；由 `feeToSetter` 通过 `setFeeTo` 设置。 |
| **权限** | `feeToSetter` 可 `setFeeTo`、`setFeeToSetter`（可移交管理员）。 |
| **查询** | `getPair`、`allPairs`、`allPairsLength`。 |

---

## UniswapV2Pair（单池 + LP 代币）

Pair 本身是 **LP 代币**（继承下面 ERC20），同时是 **恒定乘积 AMM 池**。

| 功能 | 说明 |
|------|------|
| **初始化** | `initialize(token0, token1)`：仅 Factory 在创建时调用。 |
| **加流动性** | `mint(to)`：按存入的 token0/token1 增量铸 LP；首池扣除 **MINIMUM_LIQUIDITY** 并永久锁定。 |
| **减流动性** | `burn(to)`：销毁当前合约持有的 LP，按比例取出两种代币。 |
| **兑换** | `swap(amount0Out, amount1Out, to, data)`：恒定乘积 **x·y=k**（含 **0.3%** 手续费体现在余额约束里）；支持 **flash swap**（`data` 非空时回调 `IUniswapV2Callee`）。 |
| **储备与价格** | `getReserves`；**TWAP**：`price0CumulativeLast` / `price1CumulativeLast`（按时间累加，供预言机用）。 |
| **协议费** | 当 Factory 上 `feeTo` 非零时，通过 `_mintFee` 从 **k 增长** 中铸一部分 LP 给 `feeTo`（与 Uniswap V2 文档一致）。 |
| **重入锁** | `lock`：防止重入。 |
| **工具** | `skim`：把多于储备的代币转出；`sync`：用当前余额刷新储备。 |

---

## UniswapV2ERC20（LP 代币标准功能）

| 功能 | 说明 |
|------|------|
| **ERC20** | `name` / `symbol` / `decimals`、`transfer`、`approve`、`transferFrom`、`allowance`。 |
| **EIP-2612 式 Permit** | `permit`：链下签名授权，免先 `approve` 交易（含 `DOMAIN_SEPARATOR`、`nonces`）。 |

---

## 库与数学（支撑功能）

- **`SafeMath`**：安全算术。  
- **`Math`**：如 `sqrt`（流动性计算）。  
- **`UQ112x112`**：定点数，用于价格累加器。  

---

## 测试入口（本仓库）

运行 **`yarn test`**（会先 **`yarn compile`**），用 **ethereum-waffle `MockProvider`** 对 Factory / Pair / ERC20 行为做单元测试。环境要求见 `questions/1.environment.md`。
