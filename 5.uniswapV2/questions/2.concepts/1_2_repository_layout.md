# Q1：What does the project structure look like, and what is each directory for?

## Interview-style overview (~45 seconds)

**Uniswap v2-core** is a Solidity-focused repo: **`contracts/`** holds the protocol source, **`test/`** holds TypeScript tests against a local chain, and **`build/`** (after compile) holds compiler output (ABI + bytecode) for deployment and tooling. Config at the root (**`package.json`**, **`.waffle.json`**, **`tsconfig.json`**) wires **ethereum-waffle** + **solc** for compile and **mocha** for tests.

Anything else—like **`scripts/`** or **`questions/`**—is often **project-local** (deploy helpers, notes) and is **not** part of the published `@uniswap/v2-core` npm package, which typically ships **`contracts`** and **`build`** only.

---

## Typical top-level layout

```text
v2-core/
├── contracts/          # Solidity source (the protocol)
├── test/               # TS tests (waffle / mocha)
├── build/              # Generated by yarn compile (ABI, bytecode)
├── scripts/            # Often local: Node deploy / interaction scripts
├── questions/          # Often local: learning notes (this workspace)
├── package.json
├── .waffle.json        # Waffle + solc compiler options
├── tsconfig.json       # For test/ TypeScript
├── .mocharc.json       # Mocha test runner config
└── README.md
```

## Directory roles (what to say in an interview)

| Path | Role |
|------|------|
| **`contracts/`** | **Main deliverable.** Factory, Pair, UniswapV2ERC20, plus **`interfaces/`** (external ABIs), **`libraries/`** (SafeMath, Math, UQ112x112), and sometimes **`test/`** subfolder for test-only ERC20s used in specs. |
| **`test/`** | **Behavioral verification.** Waffle `MockProvider` tests (e.g. Factory/Pair specs); **`shared/`** for test utilities. Proves core invariants without a public RPC. |
| **`build/`** | **Compiler artifacts** (`*.json` with ABI + bytecode). Created by **`yarn compile`**, consumed by deploy tooling and scripts. Often **gitignored**; republishable via npm `files` field. |
| **`scripts/`** | **Automation (optional in upstream).** Node + ethers scripts for deploy, `createPair`, liquidity, swap—**not** required to understand the protocol; useful for integration testing on testnets. |
| **`questions/`** | **Documentation / learning notes (optional).** Not part of Uniswap release; local notes and Q&A. |
| **`.waffle.json`** | **Compile pipeline:** solc version, optimizer, EVM target (`istanbul` here). |
| **`package.json`** | **Scripts:** `compile`, `test`, `deploy`, etc.; **engines** (Node); devDeps (waffle, mocha, solc, ethers, …). |

---

## How it fits the “core vs periphery” story

- **v2-core** repo = **on-chain protocol source + tests + build output.**  
- **v2-periphery** is a **different repository** (Router, etc.); you will not find Router Solidity here.
