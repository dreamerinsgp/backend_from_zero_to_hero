4. B+tree 数据结构
1）存什么数据？
存储被索引的字段值

2）怎么存数据？
Q1: 按照什么顺序存储？
示例：
CREATE TABLE People (
 last_name varchar(50) not null,
 first_name varchar(50) not null,
 dob date not null,
 key(last_name, first_name, dob)
);
在这个表中，我们创建一个索引 (last_name, first_name, dob)
排序规则：
按照字母序；在名称相同的情况下，按照下一个字段的字母序来排/日期来排

核心要点：
a: 节点插入：小于当前节点值，存放在左边；大于当前节点值，存放在右边；
b: 时间复杂度：全局扫描 O(N) ; B+tree O(logmN)   N就是数据集大小；m每个节点子节点的数量 (B+tree平衡树)


3）怎么就能够帮助我们快速的查询数据？
要点1： 
使用索引之后，存储引擎不需要全表扫描来发现目标数据；
Q1: 全表扫描是什么意思？
O(n)

1） 叶子节点和子节点的区别
叶子节点包含了指向被索引数据的指针，中间的子节点仅包含指针（不指向被索引的数据），指向它的子节点。
