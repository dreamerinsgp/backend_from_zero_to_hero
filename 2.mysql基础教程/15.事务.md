1. 定义
事务由一个或多个sql语句构成，它们以原子特性执行，也就是说，全部执行成功，才算成功；只要有一个执行失败，全部失败。 

2. 事务举例
A banking application is the classic example of why transactions are necessary.5
 Imag‐ine a bank’s database with two tables: checking and savings. To move $200 from Jane’s
checking account to her savings account, you need to perform at least three steps:
1. Make sure her checking account balance is greater than $200.
2. Subtract $200 from her checking account balance.
3. Add $200 to her savings account balance.
The entire operation should be wrapped in a transaction so that if any one of the steps
fails, any completed steps can be rolled back

在Mysql中上述例子的实现方式？ 
1 START TRANSACTION;
2 SELECT balance FROM checking WHERE customer_id = 10233276;
3 UPDATE checking SET balance = balance - 200.00 WHERE customer_id = 10233276;
4 UPDATE savings SET balance = balance + 200.00 WHERE customer_id = 10233276;
5 COMMIT;


3. 如果事务因为各种原因在执行过程中，突然中断，会发生什么问题？ 如何解决？ 

3.1） 如何解决？要求server必须满足ACID标准。
ACID: Atomicity(原子性)； Consistency(一致性)； Isolation(隔离性)； Duration(持久性)

Atomicity: it’s all or nothing

Consistency: 数据库中的状态永远是一致的 （合乎我们正常理解的认知去执行的）【守恒】 

Isolation: 是说一个事务对另一个事务通常是不可见的，直到当前的事务执行结束。 

Duration: Once committed, a transaction’s changes are permanent
