内容：
1. Read uncommitted :
又被称为dirty read （脏读） ， 它会读取到其它未提交的事务的结果

2. READ COMMITTED ：
一般数据库的默认隔离级别； 它只会读取到已提交事务的变更结果。 他会导致不可重复读 （nonrepeatable read. This means you can run the
same statement twice and see different data.） 。 

3. Repeatable read: 
存在幻读问题； 在一个事务内部，多次重复读取，会获取到相同的结果。 

4. Searializable：
通过给每行记录加锁来强制所有事务按照顺序执行，解决了幻读问题。 


小结：
1. Lower isolation levels typically allow higher concurrency and have lower overhead
2. 具体的隔离级别的选择，其实就是结合了具体业务需求下的，数据安全和并发性的取舍问题。
